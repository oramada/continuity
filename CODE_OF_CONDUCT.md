# Code Of Conduct

Continuity Protocol contributors are expected to keep collaboration respectful, technical, and safety-conscious.

## Expected Behavior

- Be direct and constructive.
- Focus criticism on code, protocol behavior, documentation, or risk.
- Respect privacy and avoid exposing sensitive identity, key, settlement, or vulnerability information.
- Treat negative attestations, abuse evidence, and trust labels as high-risk protocol areas.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Public disclosure of private keys, exploit details, or sensitive evidence.
- Misrepresenting developer-preview functionality as audited production readiness.
- Using the project to target people or organizations with unreviewed negative claims.

## Enforcement

Maintainers may moderate issues, discussions, pull requests, or contributions that violate this code. Security-sensitive issues should be moved to private reporting channels.
