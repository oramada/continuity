# Protocol Status

Continuity Protocol is a developer preview.

## Developer-Preview Stable

- Canonicalization
- Domain-separated hashes
- Ed25519 signing and verification
- Merkle inclusion proofs
- Core schema validation
- Deterministic CLI demo
- TypeScript verifier basics
- Python and Rust parity checks

## Experimental

- Trust scoring
- Graph feature vectors
- Sybil assessment
- Drift analysis
- Agent delegation
- Auditor workflows
- Service mesh operation

## Candidate Pipeline

- Production-interface ZK circuits
- Production-candidate ZK artifact manifests
- Offline settlement proof vectors
- Base mainnet dry-run checks

These are code-enforced locally but still require external review before production.

## Not Production-Ready

- Mainnet operation
- Legal or financial trust decisions
- Irreversible negative claims
- Production ZK ceremony
- Active verification-key registry
- External security or ZK audit
- Public risk labels

## Conformance Levels

`conformance:spec` is the developer-preview protocol gate.

`conformance:mainnet` is intentionally evidence-gated. It should fail until real ceremony, audit, deployment, legal, security, ops, governance, and release evidence are approved.
